# Diagram as Code 0.1.0

Initial MVP release of the self-hosted Diagram as Code toolchain.

## Included

- Fastify rendering Gateway with scoped API-key verifier records, per-principal rate limit/cache isolation, bounded concurrency and queue, timeout, output-size and media validation, SVG sanitization, structured redacted logs, aggregate metrics, weighted TTL in-memory LRU cache, cache-failure degradation, and single-flight request coalescing.
- Version-locked Gateway, Kroki fork 0.31.1, and Mermaid companion images published under the same product tag, with immutable digests in the release manifest.
- Kroki rendering for Mermaid, PlantUML/C4-PlantUML, Graphviz/DOT, and D2 with canonical SVG output and PNG where supported.
- VS Code extension with live preview, debounced render-on-save, SecretStorage API keys, explicit stable SVG export, and labeled status-bar/editor/preview/context-menu controls.
- GitHub Action with API-key and OIDC authentication that detects missing, stale, and orphaned committed SVG/PNG files without committing or commenting automatically.
- Docker Compose deployment files with an externally reachable Gateway edge network and an internal-only Kroki/Mermaid rendering network.
- SPDX SBOMs, SHA-256 checksums, a machine-readable release manifest, a four-engine pilot fixture, and upgrade/rollback runbooks.

## Requirements

- VS Code 1.100 or later for the extension.
- Docker or Podman-compatible infrastructure for the rendering stack.
- A self-hosted GitHub runner when the Gateway is available only on a private network.

## Known limitations

- The MVP recommends one Gateway replica; cache and rate-limit state are in memory.
- There is no playground, Redis/HA deployment, trusted auto-commit, Markdown fenced preview, or VS Code Web support.
- The Action updates files only in explicit `generate` mode on trusted events and never commits automatically.
- PDF, BPMN, and renderers outside Mermaid, PlantUML/C4, Graphviz/DOT, and D2 are outside the MVP.
